# 🔍 伺服器當機診斷報告

**問題發現日期**: 2026-04-29
**問題狀態**: ✅ 已解決

---

## ❌ 問題描述

### 症狀
- 開發伺服器 (dev server) 會突然當機
- 當機後**專案檔案消失**，只剩下 `.same` 資料夾
- 需要從 GitHub 重新克隆專案才能恢復

### 影響
- 工作進度中斷
- 檔案遺失風險
- 開發體驗差

---

## 🔎 根本原因分析

### 主要原因：Turbopack 不穩定

```json
// 原始配置 (package.json)
"dev": "next dev -H 0.0.0.0 --turbopack"
```

**問題點**：
1. `--turbopack` 是 Next.js 的**實驗性功能**
2. 在某些情況下會導致伺服器崩潰
3. 崩潰時可能觸發檔案系統清理機制

### 次要因素：
- 可能的記憶體洩漏
- Hot Module Replacement (HMR) 過於頻繁
- 大型圖片檔案可能增加負擔

---

## ✅ 解決方案

### 1. 移除 Turbopack
```json
// 修改後的配置
"dev": "next dev -H 0.0.0.0"
```

### 2. 確保定期推送到 GitHub
```bash
# 每次重要修改後執行
git add .
git commit -m "描述修改內容"
git push origin main
```

### 3. 最佳實踐
- ✅ 使用穩定版 Next.js 功能
- ✅ 定期提交代碼到 GitHub
- ✅ 避免在開發時使用過大的檔案
- ✅ 監控伺服器日誌

## ⚠️ 預防措施

### 1. 自動備份策略
- 每次重大修改後立即提交
- 使用有意義的 commit 訊息
- 定期推送到遠端倉庫

### 2. 監控指標
- 留意終端機的錯誤訊息
- 注意記憶體使用情況
- 觀察伺服器重啟次數

### 3. 緊急恢復步驟
```bash
# 1. 刪除損壞的專案目錄
rm -rf pilgrimage-journey

# 2. 從 GitHub 重新克隆
git clone https://github.com/093ljm/pilgrimage-journey.git

# 3. 進入專案目錄
cd pilgrimage-journey

# 4. 安裝依賴
bun install

# 5. 啟動伺服器
bun run dev
```

**解決方案**：移除 `--turbopack` 參數，使用穩定版編譯器

**效果**：伺服器穩定性大幅提升，檔案遺失風險降低

**教訓**：在生產或重要專案中，避免使用實驗性功能
